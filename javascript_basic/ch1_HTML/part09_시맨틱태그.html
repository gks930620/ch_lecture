시맨틱 태그란?

📖 정의

Semantic(시맨틱)은 "의미론적인"이라는 뜻입니다.

시맨틱 태그는 태그 자체가 의미를 가지고 있어서, 코드만 봐도 어떤 내용이 들어가는지 알 수 있는 태그입니다.

❌ Non-Semantic (의미 없음)

<div> ... </div>
<span> ... </span>

내용을 봐야만 용도를 알 수 있음

✅ Semantic (의미 있음)

<header> ... </header>
<nav> ... </nav>
<footer> ... </footer>

태그 이름만 봐도 용도를 알 수 있음

📋 주요 시맨틱 태그

<header>

페이지나 섹션의 머리글을 나타냅니다.

주로 로고, 제목, 네비게이션이 포함됩니다.

<nav>

네비게이션 링크를 포함하는 영역입니다.

메뉴, 목차 등에 사용됩니다.

<main>

문서의 주요 콘텐츠를 나타냅니다.

페이지당 하나만 사용해야 합니다!

<section>

문서의 독립적인 구획을 나타냅니다.

일반적으로 제목(h1~h6)을 포함합니다.

<article>

독립적으로 배포 가능한 콘텐츠를 나타냅니다.

블로그 글, 뉴스 기사, 댓글 등에 사용됩니다.

<aside>

메인 콘텐츠와 간접적으로 관련된 내용을 나타냅니다.

사이드바, 광고, 관련 링크 등에 사용됩니다.

<footer>

페이지나 섹션의 바닥글을 나타냅니다.

저작권 정보, 연락처, 관련 링크 등이 포함됩니다.

🏗️ 전형적인 웹 페이지 구조

<!DOCTYPE html>
<html>
<head>
  <title>시맨틱 웹</title>
</head>
<body>

  <header>
    <h1>사이트 제목</h1>
    <nav>
      <ul>
        <li><a href="#">홈</a></li>
        <li><a href="#">소개</a></li>
        <li><a href="#">연락처</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section>
      <h2>섹션 제목</h2>
      <article>
        <h3>기사 제목</h3>
        <p>기사 내용...</p>
      </article>
    </section>

    <aside>
      <h3>사이드바</h3>
      <p>관련 정보...</p>
    </aside>
  </main>

  <footer>
    <p>© 2024 My Website</p>
  </footer>

</body>
</html>

🎯 시맨틱 태그의 장점

1️⃣ 가독성 향상

코드만 봐도 구조를 쉽게 파악할 수 있습니다.

2️⃣ 유지보수 용이

의미가 명확하므로 수정과 관리가 편리합니다.

3️⃣ SEO 향상

검색 엔진이 페이지 구조를 더 잘 이해하여 검색 순위가 올라갑니다.

4️⃣ 접근성 개선

스크린 리더 등 보조 기술이 콘텐츠를 더 잘 해석할 수 있습니다.

5️⃣ 팀 협업 향상

다른 개발자가 코드를 쉽게 이해할 수 있습니다.

📌 section vs article 차이

🔷 section

주제별로 그룹화된 콘텐츠

예: 홈페이지의 "서비스 소개", "고객 후기" 등

🔷 article

독립적으로 재사용 가능한 콘텐츠

예: 블로그 포스트, 뉴스 기사, 댓글

💡 팁: article은 RSS 피드로 배포될 수 있는 콘텐츠인지 생각해보세요!

✅ 실습 과제

  1. 자신의 블로그 레이아웃을 시맨틱 태그로 구현하세요
  2. 뉴스 사이트 메인 페이지 구조를 시맨틱 태그로 만들어보세요
  3. 기존 div로 만들어진 페이지를 시맨틱 태그로 리팩토링하세요

💡 추가 정보

기타 시맨틱 태그

  • <figure>: 이미지, 도표 등
  • <figcaption>: figure 설명
  • <mark>: 강조 표시
  • <time>: 날짜/시간
  • <details>: 토글 가능한 상세정보
  • <summary>: details의 제목
