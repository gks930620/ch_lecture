## 1. 제네릭이란?

- **타입을 파라미터화**하여 컴파일 시 타입 체크
- 타입 안정성 제공
- 형변환 불필요

```java
// 제네릭 사용 전
ArrayList list = new ArrayList();
list.add("Hello");
String str = (String) list.get(0);  // 형변환 필요

// 제네릭 사용 후
ArrayList<String> list = new ArrayList<>();
list.add("Hello");
String str = list.get(0);  // 형변환 불필요
```

## 2. 제네릭 클래스

```java
public class Box<T> {
    private T item;
    
    public void setItem(T item) {
        this.item = item;
    }
    
    public T getItem() {
        return item;
    }
}

// 사용
Box<String> stringBox = new Box<>();
stringBox.setItem("Hello");
String value = stringBox.getItem();

Box<Integer> intBox = new Box<>();
intBox.setItem(100);
int number = intBox.getItem();
```

## 3. 제네릭 메소드

```java
public class GenericMethod {
    public static <T> void printArray(T[] array) {
        for (T element : array) {
            System.out.println(element);
        }
    }
    
    public static <T> T getFirst(T[] array) {
        return array[0];
    }
}

// 사용
String[] strArray = {"A", "B", "C"};
GenericMethod.printArray(strArray);

Integer[] intArray = {1, 2, 3};
Integer first = GenericMethod.getFirst(intArray);
```

## 4. 타입 파라미터 제한

```java
// extends: 상한 제한
public class NumberBox<T extends Number> {
    private T number;
    
    public void setNumber(T number) {
        this.number = number;
    }
    
    public double getDoubleValue() {
        return number.doubleValue();
    }
}

// 사용
NumberBox<Integer> intBox = new NumberBox<>();  // OK
NumberBox<Double> doubleBox = new NumberBox<>();  // OK
// NumberBox<String> stringBox = new NumberBox<>();  // 에러!
```

## 5. 와일드카드 (?)

### 5.1 <?> (제한 없음)
```java
public static void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
}
```

### 5.2 <? extends T> (상한 제한)
```java
public static double sumOfList(List<? extends Number> list) {
    double sum = 0;
    for (Number num : list) {
        sum += num.doubleValue();
    }
    return sum;
}

List<Integer> intList = Arrays.asList(1, 2, 3);
List<Double> doubleList = Arrays.asList(1.1, 2.2, 3.3);
sumOfList(intList);    // OK
sumOfList(doubleList);  // OK
```

### 5.3 <? super T> (하한 제한)
```java
public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}
```

## 6. 제네릭의 타입 소거

- 컴파일 후 제네릭 정보는 제거됨 (하위 호환성)
- 런타임에는 일반 클래스와 동일

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
// list1.getClass() == list2.getClass()  // true
```

## 7. 정리

### 핵심 개념
✅ **제네릭**: 타입을 파라미터화  
✅ **타입 안정성**: 컴파일 시 타입 체크  
✅ **형변환 불필요**: 자동 타입 변환  
✅ **extends**: 타입 제한  
✅ **와일드카드**: 유연한 타입 처리  

### 주요 타입 파라미터
- **T**: Type
- **E**: Element
- **K**: Key
- **V**: Value
- **N**: Number
