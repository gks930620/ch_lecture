## 1. 메소드란?

### 1.1 메소드의 정의
- **특정 작업을 수행하는 코드 블록**
- 코드의 재사용성과 가독성 향상
- `함수(Function)`라고도 부름

```java
접근제어자 반환타입 메소드명(매개변수) {
    // 실행 코드
    return 반환값;
}
```

## 2. 메소드 선언과 호출

### 2.1 기본 메소드
```java
// 메소드 선언
public static void sayHello() {
    System.out.println("Hello!");
}

// 메소드 호출
sayHello();  // "Hello!" 출력
```

### 2.2 매개변수가 있는 메소드
```java
public static void greet(String name) {
    System.out.println("안녕하세요, " + name + "님!");
}

greet("홍길동");  // "안녕하세요, 홍길동님!"
```

### 2.3 반환값이 있는 메소드
```java
public static int add(int a, int b) {
    return a + b;
}

int result = add(10, 20);  // 30
System.out.println(result);
```

## 3. 메소드 오버로딩

### 3.1 오버로딩이란?
- **같은 이름**의 메소드를 **여러 개** 정의
- 매개변수의 **개수**나 **타입**이 달라야 함
- 반환 타입만 다른 것은 오버로딩 아님

```java
public static int add(int a, int b) {
    return a + b;
}

public static double add(double a, double b) {
    return a + b;
}

public static int add(int a, int b, int c) {
    return a + b + c;
}

// 호출
add(10, 20);         // int 버전
add(10.5, 20.5);     // double 버전
add(10, 20, 30);     // 3개 매개변수 버전
```

## 4. 가변인자 (Varargs)

```java
public static int sum(int... numbers) {
    int total = 0;
    for (int num : numbers) {
        total += num;
    }
    return total;
}

// 호출
sum(1, 2, 3);           // 6
sum(1, 2, 3, 4, 5);     // 15
sum();                  // 0 (인자 없이도 가능)
```

## 5. 재귀 메소드

### 5.1 재귀란?
- 메소드가 **자기 자신을 호출**하는 것
- 반복문 대신 사용 가능

```java
// 팩토리얼 (5! = 5 × 4 × 3 × 2 × 1)
public static int factorial(int n) {
    if (n <= 1) {
        return 1;  // 종료 조건
    }
    return n * factorial(n - 1);  // 재귀 호출
}

factorial(5);  // 120
```

```java
// 피보나치 수열
public static int fibonacci(int n) {
    if (n <= 1) {
        return n;
    }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

fibonacci(6);  // 8 (0, 1, 1, 2, 3, 5, 8)
```

## 6. 변수의 스코프

### 6.1 지역 변수 (Local Variable)
- 메소드 내부에서 선언
- 메소드가 끝나면 소멸
- 다른 메소드에서 접근 불가

```java
public static void method1() {
    int localVar = 10;  // 지역 변수
    System.out.println(localVar);
}  // localVar 소멸

public static void method2() {
    // System.out.println(localVar);  // 에러! 접근 불가
}
```

### 6.2 전역 변수 (멤버 변수)
- 클래스 내부에서 선언
- 모든 메소드에서 접근 가능
- 객체가 소멸될 때까지 유지

```java
public class Example {
    static int globalVar = 100;  // 전역 변수 (static)
    
    public static void method1() {
        System.out.println(globalVar);  // 100
    }
    
    public static void method2() {
        globalVar = 200;
        System.out.println(globalVar);  // 200
    }
}
```

### 6.3 블록 스코프
```java
public static void main(String[] args) {
    int a = 10;
    
    if (true) {
        int b = 20;  // if 블록 내부
        System.out.println(a);  // OK
        System.out.println(b);  // OK
    }
    
    // System.out.println(b);  // 에러! b는 if 블록 안에서만 유효
}
```

## 7. Call by Value

### 7.1 Java는 Call by Value
- 메소드에 **값의 복사본**을 전달
- 원본 변수는 변경되지 않음

```java
// 기본 자료형
public static void changeValue(int x) {
    x = 100;
}

int num = 10;
changeValue(num);
System.out.println(num);  // 10 (변경 안 됨)
```

```java
// 참조 자료형 (주의!)
public static void changeArray(int[] arr) {
    arr[0] = 999;  // 배열 내용은 변경됨 (참조를 통해)
}

int[] numbers = {1, 2, 3};
changeArray(numbers);
System.out.println(numbers[0]);  // 999 (변경됨!)
```

## 8. 정리

### 핵심 개념
✅ **메소드**: 특정 작업을 수행하는 코드 블록  
✅ **매개변수**: 메소드에 전달하는 값  
✅ **반환값**: 메소드가 돌려주는 결과  
✅ **오버로딩**: 같은 이름, 다른 매개변수  
✅ **재귀**: 자기 자신을 호출  
✅ **스코프**: 변수의 유효 범위  

### 메소드 작성 팁
- 하나의 메소드는 하나의 기능만
- 메소드 이름은 동사로 시작 (예: getSum, calculateAverage)
- 매개변수는 최소화
