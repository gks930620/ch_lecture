# 상속과 다형성

## 학습 목표
- 상속의 목적과 장단점을 이해한다.
- 메소드 오버라이딩과 다형성 개념을 익힌다.
- 추상 클래스의 역할을 이해한다.

---

## 1. 상속 (Inheritance)
- 기존 클래스의 속성과 기능을 재사용하여 새 클래스를 만든다.
- 공통 코드를 부모 클래스에 두고, 자식 클래스에서 확장한다.
- "is-a" 관계에서 사용하는 것이 적절하다.

---

## 2. 오버라이딩 (Overriding)
- 자식 클래스가 부모 메소드를 재정의한다.
- 실행 시점에는 실제 객체 타입 기준으로 메소드가 선택된다.

---

## 3. 다형성 (Polymorphism)
- 부모 타입 참조 변수로 자식 객체를 다룰 수 있다.
- 공통 인터페이스 기반으로 유연한 설계를 가능하게 한다.
- 코드 결합도를 낮추고 확장성을 높인다.

---

## 4. 추상 클래스
- 공통 기능을 일부 구현하고, 일부는 추상 메소드로 강제할 수 있다.
- 객체를 직접 생성하지 않고 상속을 통해 사용한다.

---

## 5. 설계 시 주의점
- 상속은 강한 결합을 만들 수 있으므로 남용하지 않는다.
- 단순 코드 재사용 목적이면 조합(composition)도 검토한다.

---

## 정리
- 상속은 재사용 도구, 다형성은 유연성 도구다.
- 공통점은 상위 타입으로 추상화한다는 점이다.
