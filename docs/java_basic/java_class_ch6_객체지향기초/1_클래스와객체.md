# 6장. 객체지향 프로그래밍 기초

## 학습 목표
- 클래스와 객체의 차이를 이해한다.
- 필드, 생성자, 메소드의 역할을 구분한다.
- 캡슐화와 접근제어의 목적을 이해한다.

---

## 1. 클래스와 객체
- 클래스: 객체를 만들기 위한 설계도
- 객체: 클래스로 생성된 실제 인스턴스
- 하나의 클래스로 여러 객체를 만들 수 있다.

---

## 2. 클래스 구성 요소
- 필드(field): 객체의 상태(데이터)
- 생성자(constructor): 객체 초기화
- 메소드(method): 객체의 동작(행위)

---

## 3. 캡슐화
- 데이터와 동작을 하나로 묶고, 외부 접근을 제어한다.
- 보통 필드는 `private`, 접근은 getter/setter 또는 메소드로 제공한다.
- 무결성 유지와 유지보수성 향상에 유리하다.

---

## 4. static과 인스턴스
- 인스턴스 멤버: 객체마다 별도 보유
- static 멤버: 클래스 차원에서 공유
- 공통 데이터/유틸성 기능은 static이 적합하다.

---

## 정리
- 객체지향 기초의 핵심은 "상태(필드) + 행위(메소드) + 은닉(캡슐화)"이다.
- 이후 상속, 다형성, 인터페이스의 기반이 된다.
