# 인터페이스

## 학습 목표
- 인터페이스의 목적과 문법을 이해한다.
- 다중 구현과 다형성 활용 방식을 익힌다.
- 추상 클래스와의 차이를 설명할 수 있다.

---

## 1. 인터페이스란?
- 객체가 제공해야 하는 기능의 "규약(계약)"을 정의한다.
- 구현 클래스는 인터페이스 메소드를 반드시 구현해야 한다.

---

## 2. 주요 특징
- 클래스는 여러 인터페이스를 구현할 수 있다.
- 느슨한 결합(Loose Coupling)을 설계할 때 핵심 도구다.
- 기본 메소드(`default`)와 정적 메소드(`static`)를 가질 수 있다.

---

## 3. 다형성과 인터페이스
- 인터페이스 타입으로 다양한 구현체를 동일하게 다룰 수 있다.
- 구현체를 교체해도 호출 코드 변경을 최소화할 수 있다.

---

## 4. 추상 클래스와 차이
- 인터페이스: "무엇을 할 수 있는가" 중심
- 추상 클래스: "무엇을 공유하는가" 중심
- 실무에서는 둘을 목적에 따라 병행 사용한다.

---

## 5. 익명 객체와 함수형 인터페이스
- 일회성 구현은 익명 객체로 작성할 수 있다.
- 추상 메소드가 1개인 인터페이스는 함수형 인터페이스로 람다와 결합된다.

---

## 정리
- 인터페이스는 확장성과 테스트 용이성을 높이는 핵심 추상화 수단이다.
