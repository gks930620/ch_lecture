# 11장. 제네릭 (Generics)

## 학습 목표
- 제네릭의 목적을 이해한다.
- 타입 파라미터 문법을 익힌다.
- 제한된 타입, 와일드카드 활용 개념을 익힌다.

---

## 1. 제네릭이 필요한 이유
- 컴파일 시 타입 안정성을 높인다.
- 불필요한 형변환(casting)을 줄인다.
- 컬렉션과 유틸 코드의 재사용성을 높인다.

---

## 2. 기본 문법
- 타입 파라미터: `<T>`, `<K, V>` 등
- 제네릭 클래스, 제네릭 메소드 정의 가능
- 관례적으로 단일 대문자를 타입 파라미터로 사용

---

## 3. 타입 제한
- `<T extends SomeType>` 형태로 상한 제한 가능
- 특정 타입 계층에서만 동작하도록 제약을 건다.

---

## 4. 와일드카드
- `?` : 임의 타입
- `? extends T` : T 또는 하위 타입 (읽기 중심)
- `? super T` : T 또는 상위 타입 (쓰기 중심)

---

## 5. 주의점
- 타입 소거(Type Erasure)로 런타임에는 제네릭 타입 정보가 제한된다.
- 기본형은 직접 타입 파라미터로 쓸 수 없고 래퍼 타입을 사용한다.

---

## 정리
- 제네릭은 "재사용"보다 "타입 안정성" 확보 관점에서 이해하는 것이 정확하다.
