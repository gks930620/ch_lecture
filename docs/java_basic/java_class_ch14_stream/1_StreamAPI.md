# Stream API

## 학습 목표
- Stream의 처리 모델을 이해한다.
- 중간 연산과 최종 연산을 구분한다.
- 선언형 데이터 처리의 장단점을 파악한다.

---

## 1. Stream 개요
- 컬렉션 데이터를 선언형으로 처리하는 API
- 원본 데이터를 직접 변경하지 않고, 연산 파이프라인으로 처리한다.

---

## 2. 처리 단계
- 스트림 생성: `stream()`, `Arrays.stream()` 등
- 중간 연산: `filter`, `map`, `sorted`, `distinct`
- 최종 연산: `forEach`, `collect`, `reduce`, `count`, `anyMatch`

---

## 3. 핵심 특징
- 지연 연산(Lazy Evaluation)
- 내부 반복(Internal Iteration)
- 함수형 인터페이스와 결합해 간결한 코드 작성 가능

---

## 4. 주의점
- 스트림은 재사용할 수 없다.
- 부작용(side effect)이 큰 연산은 피하는 것이 좋다.
- 병렬 스트림은 성능 이점이 항상 보장되지는 않는다.

---

## 5. 실무 가이드
- 읽기 쉬운 파이프라인을 우선한다.
- 복잡한 체인은 중간 변수나 메서드 분리로 명확히 한다.

---

## 정리
- Stream API는 "데이터 처리 흐름"을 코드로 표현하는 도구다.
