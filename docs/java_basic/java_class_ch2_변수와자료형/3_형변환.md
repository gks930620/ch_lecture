# 2.3 형변환 (Casting)

## 학습 목표
- 자동 형변환과 강제 형변환의 차이를 구분한다.
- 숫자 타입 변환 시 데이터 손실 가능성을 이해한다.
- 연산에서 발생하는 타입 승격 규칙을 익힌다.

---

## 1. 자동 형변환 (묵시적 변환)
- 작은 범위 타입이 큰 범위 타입으로 변환될 때 자동으로 일어난다.
- 예: `int` -> `long`, `float` -> `double`

---

## 2. 강제 형변환 (명시적 변환)
- 큰 범위 타입을 작은 범위 타입으로 바꿀 때 개발자가 직접 캐스팅한다.
- 예: `long` -> `int`, `double` -> `int`
- 값 손실, 오버플로우 가능성을 반드시 고려해야 한다.

---

## 3. 연산 시 타입 규칙
- 정수 연산은 보통 `int` 이상으로 승격된다.
- 정수와 실수의 연산 결과는 실수형으로 승격된다.

---

## 4. 문자와 숫자 변환
- `char`는 내부적으로 유니코드 정수값을 가진다.
- 문자 연산 시 정수로 처리되는 경우가 있다.

---

## 정리
- 형변환은 문법보다 데이터 보존 관점이 핵심이다.
- 변환이 필요한 이유와 손실 가능성을 항상 같이 점검해야 한다.
