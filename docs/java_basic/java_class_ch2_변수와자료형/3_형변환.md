# 형변환 (Casting)

## 학습 목표
- Java의 자동 형변환(확대 변환)과 강제 형변환(축소 변환)을 구분할 수 있다.
- 연산 과정에서 발생하는 타입 승격 규칙을 이해하고 결과 타입을 예측할 수 있다.
- primitive 변환, 참조형 캐스팅, 오토박싱/언박싱의 위험 지점을 설명할 수 있다.

---

## 1. 형변환이 필요한 이유

프로그램에서는 여러 타입의 값이 함께 사용된다.

- 저장 용량이 다른 정수 타입 사이 이동
- 정수와 실수 혼합 연산
- 부모/자식 타입 객체 다루기
- API 입력 타입 맞추기 (`String` -> `int` 등)

형변환은 단순 문법이 아니라 **데이터 보존 여부**와 연결된다.

---

## 2. primitive 형변환 기본 규칙

## 2.1 자동 형변환 (Widening)

작은 범위에서 큰 범위로 갈 때는 자동 변환된다.

예:
- `byte -> short -> int -> long -> float -> double`
- `char -> int -> long -> float -> double`

```java
int i = 100;
long l = i;      // 자동
double d = l;    // 자동
```

자동 형변환은 "값 범위를 담을 수 있는 방향"으로만 일어난다.

### 2.2 강제 형변환 (Narrowing)

큰 범위에서 작은 범위로 갈 때는 명시적 캐스팅이 필요하다.

```java
long big = 3_000_000_000L;
int small = (int) big; // 축소 변환
System.out.println(small); // 값 손실 가능
```

축소 변환에서는 다음 위험이 있다.
- 상위 비트 손실
- 오버플로우/언더플로우
- 소수점 버림 (`double -> int`)

---

## 3. 정수/실수 변환의 실제 동작

## 3.1 `double -> int`

```java
double pi = 3.99;
int n = (int) pi; // 3
```

- 반올림이 아니라 소수부 절삭(버림)이다.

### 3.2 반올림이 필요하면

```java
long rounded = Math.round(pi); // 4
```

단순 캐스팅과 반올림은 목적이 다르므로 혼동하면 안 된다.

### 3.3 `long -> float`의 정밀도 손실

`float`는 24비트 수준의 유효 정밀도를 가지므로 큰 정수를 정확히 못 담을 수 있다.

```java
long x = 16_777_217L;      // 2^24 + 1
float f = x;
System.out.println((long) f); // 16_777_216 가능
```

## 4. 연산 시 타입 승격 규칙 (매우 중요)

Java에서 연산은 피연산자 타입 그대로 처리되지 않는 경우가 많다.

## 4.1 정수 승격 (Unary/Binary Numeric Promotion)

`byte`, `short`, `char`는 산술 연산 전에 `int`로 승격된다.

```java
byte a = 10;
byte b = 20;
// byte c = a + b; // 컴파일 오류
int c = a + b;     // 결과 타입은 int
```

### 4.2 혼합 연산 결과 타입

연산에 `double`이 있으면 결과는 `double`,  
그다음 `float`, `long`, `int` 순으로 결정된다.

```java
int i = 10;
double d = 3.5;
double r = i + d; // double
```

### 4.3 정수 나눗셈 함정

```java
int a = 5;
int b = 2;
double x = a / b;       // 2.0 (이미 int 나눗셈이 끝난 뒤 대입)
double y = (double)a / b; // 2.5
```

---

## 5. 컴파일 타임 상수와 대입 허용 규칙

다음은 캐스팅 없이도 허용된다.

```java
byte b = 100; // 리터럴이 byte 범위 안
```

하지만 범위를 벗어나면 오류다.

```java
// byte b = 200; // 컴파일 오류
```

또한 `final` 상수는 컴파일 타임에 계산되어 대입이 허용될 수 있다.

```java
final int N = 10;
byte b = N; // 가능
```

---

## 6. `char` 관련 변환

`char`는 0~65535 범위의 unsigned 16비트 값으로 다뤄진다.

```java
char ch = 'A';
int code = ch;         // 65
char next = (char) (ch + 1); // 'B'
```

`ch + 1` 결과는 `int`이므로 `char` 대입 시 캐스팅이 필요하다.

---

## 7. 참조형 캐스팅: 업캐스팅/다운캐스팅

## 7.1 업캐스팅 (자동)

자식 객체를 부모 타입 참조로 다루는 것은 자동 변환된다.

```java
Object obj = "hello"; // String -> Object
```

### 7.2 다운캐스팅 (명시적)

부모 참조를 자식 타입으로 되돌릴 때는 명시적 캐스팅 필요:

```java
Object obj = "hello";
String s = (String) obj; // 가능
```

실제 객체 타입이 다르면 런타임에서 `ClassCastException` 발생:

```java
Object obj = 123;
String s = (String) obj; // 예외
```

안전하게 처리:

```java
if (obj instanceof String str) {
    System.out.println(str.length());
}
```

---

## 8. 오토박싱/언박싱과 함정

primitive와 Wrapper 타입 사이에는 자동 변환이 일어난다.

```java
Integer a = 10; // boxing
int b = a;      // unboxing
```

주의점:
- `Integer x = null; int y = x;` -> `NullPointerException`
- Wrapper 비교 시 `==`는 참조 비교가 될 수 있음

값 비교는 `equals` 또는 primitive로 변환 후 비교한다.

---

## 9. 문자열과 숫자 변환

입력 데이터 처리에서 자주 사용된다.

```java
int n = Integer.parseInt("123");
double d = Double.parseDouble("3.14");
String s = String.valueOf(100);
```

잘못된 형식:

```java
Integer.parseInt("12a"); // NumberFormatException
```

실무에서는 예외 처리를 함께 작성해야 안정적이다.

---

## 10. 데이터 손실을 줄이는 실무 전략

1. 축소 변환 전 범위 체크
2. 금액 계산은 `double` 대신 `BigDecimal` 검토
3. 캐스팅을 줄이고 API 타입을 일관되게 설계
4. 경계값 테스트(`MIN_VALUE`, `MAX_VALUE`)를 자동화
5. 복합 대입 연산의 숨은 캐스팅 동작 이해

복합 대입 예:

```java
short s = 1;
s += 1;        // 컴파일 가능(내부 캐스팅)
// s = s + 1;  // 컴파일 오류(int 결과)
```

---

## 11. 정리

- 형변환의 핵심은 "변환 가능 여부"보다 "정보 손실 여부"다.
- 연산 승격 규칙을 이해하면 타입 오류와 의도치 않은 결과를 크게 줄일 수 있다.
- 참조형 캐스팅, 박싱/언박싱, 문자열 파싱까지 함께 이해해야 실무 코드에서 안전하다.
