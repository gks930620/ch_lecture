# 2.2 참조 자료형 - String과 배열

## 학습 목표
- 기본 자료형과 참조 자료형의 차이를 이해한다.
- `String`의 불변 특성을 이해한다.
- 배열의 구조와 사용법을 익힌다.

---

## 1. 참조 자료형이란?
- 값 자체가 아니라 객체의 위치(참조값)를 변수에 저장한다.
- 대표 예: `String`, 배열, 사용자 정의 클래스

---

## 2. String
- 문자열을 다루는 클래스
- Java에서 문자열 리터럴은 문자열 풀(String Pool)에서 관리될 수 있다.
- `String`은 불변(immutable) 객체이므로 변경 시 새 객체가 생성된다.

---

## 3. 배열(Array)
- 같은 타입의 데이터를 연속된 공간에 저장한다.
- 인덱스는 `0`부터 시작한다.
- 길이는 생성 후 변경할 수 없다.

---

## 4. 자주 하는 실수
- 배열 범위를 벗어나 접근하면 예외가 발생한다.
- `null` 참조를 사용하면 `NullPointerException`이 발생한다.
- `String` 비교를 `==`로 하면 의도와 다를 수 있어 `equals()`를 사용해야 한다.

---

## 정리
- 참조 자료형은 "객체를 가리킨다"는 관점으로 이해해야 한다.
- `String`의 불변성과 배열의 고정 길이는 실무에서도 매우 중요하다.
