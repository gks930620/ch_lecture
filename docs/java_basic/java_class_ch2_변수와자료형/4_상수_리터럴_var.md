# 2.4 상수, 리터럴, var

## 학습 목표
- 상수(`final`)의 의미를 이해한다.
- 리터럴의 종류와 표기법을 정리한다.
- `var`의 동작 원리와 사용 범위를 이해한다.

---

## 1. 상수 (`final`)
- 한 번 값이 대입되면 변경할 수 없는 변수
- 의미가 고정된 값(세율, 설정값 등)에 사용
- 관례적으로 상수명은 대문자 + 밑줄(`MAX_SIZE`) 형태 사용

---

## 2. 리터럴 (Literal)
- 코드에 직접 적는 값 자체
- 예: 정수 리터럴, 실수 리터럴, 문자 리터럴, 문자열 리터럴, 불리언 리터럴
- 숫자 리터럴은 가독성을 위해 `_` 구분자를 사용할 수 있다.

---

## 3. `var` (지역 변수 타입 추론)
- 컴파일러가 초기값을 보고 타입을 추론한다.
- 지역 변수에서만 사용 가능하다.
- 타입이 사라지는 것이 아니라 "추론"될 뿐이다.

---

## 4. 사용 가이드
- 타입이 명확할 때만 `var`를 사용한다.
- 코드 가독성이 떨어지는 경우에는 명시 타입을 사용한다.
- `final`과 `var`를 함께 사용해 불변 지역 변수로 선언할 수 있다.

---

## 정리
- `final`은 값 안정성, 리터럴은 값 표현, `var`는 선언 간결성이 핵심이다.
- 세 기능을 구분해서 사용하면 코드 품질이 높아진다.
