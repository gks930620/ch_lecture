# 12장. 컬렉션 프레임워크

## 학습 목표
- 컬렉션 프레임워크의 구조를 이해한다.
- `List`, `Set`, `Map`의 차이를 구분한다.
- 상황에 맞는 컬렉션 선택 기준을 익힌다.

---

## 1. 컬렉션이란?
- 여러 데이터를 관리하기 위한 표준 자료구조 API
- 배열보다 유연하고 기능이 풍부하다.

---

## 2. 주요 인터페이스
- `List`: 순서 있음, 중복 허용
- `Set`: 순서 보장 선택적, 중복 불가
- `Map`: 키-값 쌍 저장, 키 중복 불가

---

## 3. 대표 구현체
- `ArrayList`: 조회 빠름, 중간 삽입/삭제 비용 큼
- `LinkedList`: 중간 삽입/삭제 유리, 인덱스 조회 비용 큼
- `HashSet`: 빠른 중복 제거/검색
- `HashMap`: 빠른 키 기반 조회

---

## 4. 정렬과 비교
- 자연 정렬: `Comparable`
- 사용자 정의 정렬: `Comparator`
- `equals()`/`hashCode()` 계약은 `Set`/`Map`에서 특히 중요

---

## 5. 실무 팁
- 선언은 인터페이스 타입으로, 생성은 구현체로 한다.
- 동시성 환경에서는 동기화 컬렉션/동시성 컬렉션을 고려한다.

---

## 정리
- 컬렉션 선택 기준은 "중복 허용 여부, 순서, 조회/수정 패턴"이다.
