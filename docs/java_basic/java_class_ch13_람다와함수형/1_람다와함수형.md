# 13장. 람다와 함수형 프로그래밍

## 학습 목표
- 람다식의 문법과 목적을 이해한다.
- 함수형 인터페이스 개념을 익힌다.
- 메소드 참조와 기본 함수형 인터페이스를 활용할 수 있다.

---

## 1. 람다식이란?
- 익명 함수 형태로 동작을 값처럼 전달하는 문법
- 코드량을 줄이고 의도를 간결하게 표현한다.

---

## 2. 함수형 인터페이스
- 추상 메소드가 하나인 인터페이스
- `@FunctionalInterface`로 명시 가능
- 람다식은 함수형 인터페이스 구현체로 동작한다.

---

## 3. 자주 쓰는 함수형 인터페이스
- `Consumer<T>`: 입력 소비
- `Supplier<T>`: 값 공급
- `Function<T, R>`: 변환
- `Predicate<T>`: 조건 검사

---

## 4. 메소드 참조
- 람다식을 더 간결하게 표현하는 문법
- `클래스명::정적메소드`, `객체참조::인스턴스메소드` 등

---

## 5. 실무 포인트
- 람다는 짧고 명확할 때만 유지보수성이 높다.
- 복잡한 로직은 람다 안에 넣기보다 메소드로 분리한다.

---

## 정리
- 람다는 "문법 단축"이 아니라 "동작 전달" 관점으로 이해해야 한다.
