# 예외 처리 (Exception Handling)

## 학습 목표
- 예외의 개념과 필요성을 이해한다.
- 체크 예외와 언체크 예외를 구분한다.
- `try-catch-finally`, `throws`, 사용자 정의 예외를 익힌다.

---

## 1. 예외란?
- 프로그램 실행 중 발생하는 비정상 상황
- 예외 처리는 프로그램 중단을 줄이고 안정성을 높인다.

---

## 2. 예외 분류
- 체크 예외(Checked): 컴파일러가 처리 강제
- 언체크 예외(Unchecked): 런타임에 발생, 선택적 처리

---

## 3. 예외 처리 방식
- `try-catch`: 예외를 잡아서 처리
- `finally`: 예외 여부와 관계없이 항상 실행
- `throws`: 호출한 쪽으로 예외를 넘김

---

## 4. 사용자 정의 예외
- 도메인 규칙 위반을 명확히 표현할 때 사용
- 예외 이름만으로도 문제 원인을 파악하기 쉬워진다.

---

## 5. 실무 팁
- 예외를 무조건 숨기지 말고 의미 있는 메시지로 변환한다.
- 자원 정리는 `try-with-resources`를 우선 사용한다.
- 정상 흐름 제어를 예외로 처리하지 않는다.

---

## 정리
- 예외 처리는 오류 "회피"가 아니라 오류 "관리"다.
- 적절한 위치에서 잡고, 필요한 정보와 함께 전파하는 것이 핵심이다.
