# 입출력 (I/O)

## 학습 목표
- Java 입출력의 기본 흐름을 이해한다.
- 바이트 스트림과 문자 스트림의 차이를 구분한다.
- 파일 읽기/쓰기 시 예외 처리와 자원 관리를 익힌다.

---

## 1. I/O 개념
- 입력(Input): 외부에서 프로그램으로 데이터가 들어옴
- 출력(Output): 프로그램에서 외부로 데이터가 나감
- 파일, 네트워크, 메모리 등이 대상이 될 수 있다.

---

## 2. 스트림 종류
- 바이트 스트림: `InputStream`, `OutputStream`
- 문자 스트림: `Reader`, `Writer`
- 텍스트 처리에는 문자 스트림이 일반적으로 더 적합하다.

---

## 3. 버퍼링
- `BufferedInputStream`, `BufferedReader` 등으로 성능 향상
- 작은 I/O를 여러 번 수행하는 비용을 줄인다.

---

## 4. 자원 관리
- 파일/소켓은 사용 후 반드시 닫아야 한다.
- `try-with-resources`를 사용하면 자동으로 자원이 정리된다.

---

## 5. 실무 포인트
- 인코딩(UTF-8 등)을 명시해 문자 깨짐을 방지한다.
- 예외 발생 시 부분 저장/손상 가능성을 고려해 처리 전략을 세운다.

---

## 정리
- I/O 핵심은 "스트림 선택 + 버퍼링 + 안전한 자원 정리"다.
