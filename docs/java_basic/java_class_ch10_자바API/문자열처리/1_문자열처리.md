# 문자열 처리

## 학습 목표
- `String`의 특성과 주요 메소드를 이해한다.
- 문자열 비교와 변환의 기본 패턴을 익힌다.
- `StringBuilder`의 필요성을 이해한다.

---

## 1. String 기본
- `String`은 불변(immutable) 객체다.
- 변경처럼 보이는 연산도 실제로는 새 문자열을 만든다.

---

## 2. 자주 쓰는 메소드
- 길이/검색: `length()`, `indexOf()`, `contains()`
- 추출/분리: `substring()`, `split()`
- 비교: `equals()`, `equalsIgnoreCase()`
- 정리: `trim()`, `replace()`, `toLowerCase()`, `toUpperCase()`

---

## 3. 문자열 비교 원칙
- 내용 비교는 `equals()`
- 참조 비교는 `==` (의도하지 않게 쓰지 않도록 주의)

---

## 4. StringBuilder
- 반복 연결이 많은 경우 `String`보다 효율적
- `append()`, `insert()`, `delete()`, `reverse()` 등을 제공

---

## 5. 실무 포인트
- 사용자 입력 문자열은 공백/대소문자 정규화 후 비교한다.
- 로그나 대량 텍스트 조립은 `StringBuilder`를 우선 고려한다.

---

## 정리
- 문자열 처리는 기능 사용법보다 "불변 객체" 특성을 이해하는 것이 핵심이다.
