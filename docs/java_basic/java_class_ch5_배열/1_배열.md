# 배열 (Array)

## 학습 목표
- 배열의 개념과 필요성을 이해한다.
- 1차원/다차원 배열 사용법을 익힌다.
- 배열 처리 시 자주 발생하는 오류를 예방한다.

---

## 1. 배열의 개념
- 같은 타입의 데이터를 순서대로 저장하는 자료구조
- 인덱스로 요소에 빠르게 접근할 수 있다.
- 길이는 생성 시 고정된다.

---

## 2. 배열 선언과 생성
- 선언: 배열 참조 변수를 만든다.
- 생성: 실제 저장 공간을 할당한다.
- 초기값 규칙: 숫자 `0`, 논리 `false`, 참조 `null`

---

## 3. 배열 순회
- 기본 `for`문으로 인덱스 기반 처리
- 향상된 `for-each`로 값 중심 순회
- 인덱스 제어가 필요하면 기본 `for`가 유리

---

## 4. 다차원 배열
- 배열 안에 배열이 들어가는 구조
- 표 형태 데이터 처리에 적합
- 행/열 길이가 다를 수 있는 가변 배열도 가능

---

## 5. 실수 방지 포인트
- `ArrayIndexOutOfBoundsException` 주의
- `null` 배열 접근 주의
- 복사는 얕은 복사/깊은 복사를 구분해야 한다.

---

## 정리
- 배열은 가장 기본적인 데이터 저장 구조다.
- 길이 고정과 인덱스 기반 접근 특성을 정확히 이해하면 이후 컬렉션 학습도 쉬워진다.
