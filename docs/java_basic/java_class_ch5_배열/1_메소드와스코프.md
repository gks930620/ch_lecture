# 5장. 메소드와 스코프

## 학습 목표
- 메소드의 역할과 구성 요소를 이해한다.
- 매개변수와 반환값의 의미를 구분한다.
- 변수의 유효 범위(scope)를 이해한다.

---

## 1. 메소드란?
- 특정 기능을 하나로 묶은 실행 단위
- 중복 코드를 줄이고 재사용성을 높인다.

구성 요소:
- 접근제어자, 반환 타입, 메소드명, 매개변수, 본문

---

## 2. 매개변수와 반환값
- 매개변수: 호출 시 전달받는 입력값
- 반환값: 실행 결과를 호출한 쪽으로 돌려주는 값
- 반환값이 없으면 `void` 사용

---

## 3. 스코프(scope)
- 지역 변수: 메소드(또는 블록) 내부에서만 유효
- 필드(인스턴스 변수): 객체 생명주기 동안 유효
- 정적 변수(static): 클래스 로딩 시점부터 프로그램 종료까지 유효

---

## 4. 자주 하는 실수
- 지역 변수를 초기화하지 않고 사용
- 블록 밖에서 블록 내부 변수를 참조
- 메소드가 여러 책임을 가져 지나치게 길어짐

---

## 정리
- 메소드는 기능 분리, 스코프는 변수 생명주기 관리 도구다.
- 작고 명확한 메소드가 유지보수에 가장 유리하다.
