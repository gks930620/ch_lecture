# 멀티스레드

## 학습 목표
- 프로세스와 스레드의 차이를 이해한다.
- Java 스레드 생성/실행 방법을 익힌다.
- 동기화와 동시성 문제의 핵심을 이해한다.

---

## 1. 스레드란?
- 프로세스 내부에서 실행되는 작업 단위
- 멀티스레드는 동시에 여러 작업을 처리해 응답성을 높인다.

---

## 2. 생성 방식
- `Thread` 클래스 상속
- `Runnable` 구현
- 실무에서는 보통 `ExecutorService` 기반으로 관리한다.

---

## 3. 동시성 문제
- 경쟁 상태(Race Condition)
- 가시성(Visibility) 문제
- 원자성(Atomicity) 문제

---

## 4. 해결 기법
- `synchronized` 블록/메서드
- `volatile` 키워드
- `Lock`, `Atomic` 클래스
- 스레드 안전 컬렉션 활용

---

## 5. 실무 팁
- 공유 상태를 줄이는 설계가 가장 강력한 해법이다.
- 스레드 수를 무작정 늘리기보다 작업 특성(CPU/I/O)을 먼저 분석한다.
- 예외 처리와 종료 제어를 반드시 포함한다.

---

## 정리
- 멀티스레드의 핵심은 "병렬성"이 아니라 "안전한 동시성 제어"다.
